Privacy Policy

Effective Date: June 1, 2026  |  Last Updated: June 1, 2026

1. Overview

The CX Agent Studio Google Workspace Add-on ("Add-on") is developed and maintained by Yash Kavaiya (@genai_guru). This Privacy Policy explains what data the Add-on accesses, how it is used, and your rights as a user.

2. Data We Access

The Add-on requests the following Google Workspace permissions solely to provide its functionality:

  • gmail.readonly — Read email metadata to provide context for CX Agent Studio workflows. The Add-on never reads email body content beyond what you explicitly select.
  • drive.readonly — Read Drive files you select (e.g. JSON agent configs) to run lint checks.
  • spreadsheets.readonly — Read Sheets data you select for running evaluation test cases.
  • documents.readonly — Read Docs content you select for contextual agent operations.
  • userinfo.email / userinfo.profile — Identify the signed-in user to personalise the Add-on experience.
  • script.external_request — Make outbound HTTP calls to the Add-on's backend service (hosted on Google Cloud Run within your own GCP project).
  • script.storage — Store your settings (backend URL, project ID) locally in Apps Script user properties.

3. Data We Do NOT Collect

  • We do not collect, store, or transmit your email content to any third-party server.
  • We do not sell, rent, or share your data with any third party.
  • We do not track usage analytics beyond what Google Workspace and Cloud Logging provide within your own GCP project.
  • We do not store credentials, tokens, or personal data outside of your own Google user properties and your own GCP project.

4. Backend Service

The Add-on communicates with a Cloud Run backend service that you deploy into your own Google Cloud project (gen-ai-guru-gdg-pune by default). All data sent to this backend stays within your GCP project. The backend uses the cxas-scrapi library to interact with CX Agent Studio / Dialogflow CX APIs under your credentials.

5. Data Retention

Settings stored via Apps Script user properties (backend URL, project ID) are retained until you clear them via the Add-on Settings panel or uninstall the Add-on. No personal data is retained on any external server.

6. Security

All communication between the Add-on and the backend is authenticated using Google Identity tokens (OIDC). The Cloud Run service is configured to require authentication and does not allow unauthenticated requests.

7. Children's Privacy

This Add-on is not directed at children under 13. We do not knowingly collect data from children.

8. Changes to This Policy

We may update this policy. The "Last Updated" date at the top reflects the latest revision. Continued use of the Add-on constitutes acceptance of the updated policy.

9. Google API Services User Data Policy

The Add-on's use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.
